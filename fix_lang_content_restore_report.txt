多语言「俄语内容覆盖」问题修复报告
================================
时间: 2026-05-27

一、问题诊断
------------
1. 扫描结论：/pl/、/de/、/fr/ 等目录的 index.html 等核心页**并非**被俄语文件整体替换
   （各语言 html lang、导航文案、标题均正确：如 pl=DO DOMU，de=ZU HAUSE，ru=Главная）
2. 用户感知「全是俄语」的**主因**：
   上一轮「语言隔离」脚本误将右上角 ct-language-switcher 中
   `../de/index.html`、`../pl/index.html` 等改成了 `index.html`
   → 在 /ru/index.html 点击「波兰语」仍打开 /ru/index.html（俄语首页）
   → 在 /pl/ 点击「德语」仍打开 /pl/index.html（波兰语首页）

二、已执行修复
--------------
脚本：_fix_lang_isolation_restore.mjs

| 项目 | 数量 |
|------|------|
| 语言菜单链接恢复 | 12,906 处 |
| hreflang 栏目页校正 | 多页（news/firm-news/industry-news 等） |
| NPACKPM_LANG_HOMES 按目录深度 | 1,007 文件 |

| 修改文件 | ~1,000 |

------------------
• /ru/index.html：波兰语 → `../pl/index.html`，德语 → `../de/index.html`
• /pl/index.html：德语 → `../de/index.html`，英语 → `../index.html`
• /pl/wiadomosci-branzowe.html：hreflang 指向 `../pl/wiadomosci-branzowe.html`（非 index）
